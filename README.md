# 📄 Reprography System – Core Module


**Репрография документов** – локальный бэкенд-сервис для загрузки, хранения, поиска и выдачи изображений (JPEG, PNG, GIF, TIFF).
Построен на **чистом PHP** и **PostgreSQL** с максимальным использованием возможностей СУБД (JSONB, полнотекстовый поиск, триггеры, индексы).

---

## 🎯 Цели этапа разработки

- Создать **надёжное ядро** (CRUD + загрузка/выдача изображений) без фреймворков.
- Использовать **BYTEA** для хранения бинарных данных и **JSONB** для метаданных.
- Реализовать **полнотекстовый поиск** по имени и описанию.
- Обеспечить **простоту развёртывания** и масштабируемость (в будущем – очередь задач, кеширование).

---

## 🧰 Технологии

| Компонент      | Технология                        |
|----------------|-----------------------------------|

## 🚀 План действий (этапы разработки)

1. **✅ Создание core-модуля** – выполнено (класс `Core` с методами upload, get, update, delete, list, getImage).
2. **✅ Простейший API-роутер** – реализован в `index.php` (эндпоинты `/documents`, `/documents/{id}`, `/documents/{id}/image`).
3. **🔜 Создание схемы БД** – написать SQL-скрипт для таблицы `documents` (поля: id, filename, image_data, thumb_data, mime_type, file_size, width, height, metadata JSONB, description, search_vector, uploaded_at, updated_at; индексы, триггер).
4. **🔜 Настройка окружения** – подготовка конфигурации подключения (переменные окружения или конфиг-файл).
5. **🔜 Тестирование** – ручное тестирование всех эндпоинтов (Postman/curl).
6. **🔜 Оптимизация** – добавить кеширование HTTP (ETag, Last-Modified), потоковую выдачу для больших файлов.
7. **🔜 Фоновая обработка** – вынести генерацию превью и извлечение EXIF в очередь на основе PostgreSQL (таблица `jobs`).
8. **🔜 Расширение** – поддержка Large Objects для файлов > 5 МБ, конвертация в другие форматы, наложение водяных знаков.

## 🛠️ Быстрый старт (локальная разработка)

### 1. Клонирование репозитория
```bash
git clone git@github.com:testbybyscan-del/reprography.git
cd reprography
```

### 2. Настройка PostgreSQL

- Создать базу данных: `CREATE DATABASE repro;`
- Создать пользователя и дать права.
- Выполнить SQL-скрипт создания таблицы (будет добавлен позже).

### 3. Конфигурация

В `core.php` или отдельном конфиге указать параметры подключения:

```php
$core = new Core('localhost', 5432, 'repro', 'app_user', 'secret');
```

### 4. Запуск PHP-сервера

```bash
php -S localhost:8000
```

API будет доступен по адресу `http://localhost:8000/index.php`.
