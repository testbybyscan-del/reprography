# 📄 Reprography System – Core Module


[![PHP](https://img.shields.io/badge/PHP-8.2-777BB4?logo=php&logoColor=white)](https://php.net/)
[![PostgreSQL](https://img.shields.io/badge/PostgreSQL-15-4169E1?logo=postgresql&logoColor=white)](https://postgresql.org/)

**Кратко:** Локальный бэкенд-сервис для репрографии документов (изображения + в будущем текстовые документы). Реализован на чистом PHP, данные хранятся в PostgreSQL (BYTEA, JSONB, полнотекстовый поиск). Готов к разработке и развёртыванию в Docker.

## 🚀 API Эндпоинты

| Метод | URL | Описание | Пример ответа |
|-------|-----|----------|---------------|
| `POST` | `/documents` | Загрузить изображение (multipart/form-data) | `{"id": 1}` |
| `GET` | `/documents` | Список документов (пагинация, фильтры, поиск) | `[{...}, {...}]` |
| `GET` | `/documents/{id}` | Метаданные документа | `{"id":"1","filename":"...",...}` |
| `PUT` | `/documents/{id}` | Обновить описание или метаданные (JSON) | – (204) |
| `DELETE` | `/documents/{id}` | Удалить документ | – (204) |
| `GET` | `/documents/{id}/image` | Оригинальное изображение | бинарный поток |
| `GET` | `/documents/{id}/thumb` | Превью (миниатюра) | бинарный поток |
| `GET` | `/stats` | Статистика по документам | `{"total_docs":"1",...}` |

**Параметры GET /documents:**
- `search` – строка для полнотекстового поиска
- `from` / `to` – фильтр по дате (Y-m-d)
- `metadata` – JSON-объект с фильтром по полям (например, `{"author":"Bybyscan"}`)
- `limit` – количество записей (макс. 100)
- `offset` – смещение для пагинации

## 📋 Примеры curl-запросов

```bash
# Загрузка изображения
curl -X POST http://localhost:8080/index.php/documents \
  -F "image=@/path/to/image.jpg" \
  -F "description=Мой снимок" \
  -F "metadata={\"author\":\"Bybyscan\",\"location\":\"Studio\"}"

# Получить список с поиском
curl "http://localhost:8080/index.php/documents?search=снимок&limit=10&offset=0"

# Получить метаданные
curl http://localhost:8080/index.php/documents/1

# Скачать оригинал
curl http://localhost:8080/index.php/documents/1/image --output original.jpg

# Скачать превью
curl http://localhost:8080/index.php/documents/1/thumb --output thumb.jpg

# Получить статистику
curl http://localhost:8080/index.php/stats
```

## 🐳 Быстрый старт (Docker)

1. **Клонировать репозиторий**
   
   ```bash
   git clone git@github.com:testbybyscan-del/reprography.git
   cd reprography
   ```
2. **Создать .env из .env.example** и отредактировать при необходимости:
   
   ```bash
   cp .env.example .env
   ```
3. **Запустить контейнеры**
   
   ```bash
   docker compose up -d --build
   ```
4. **Проверить работу**
   
   ```bash
   curl http://localhost:8080/index.php/documents
   ```
   
   (должен вернуть `[]` или список)
5. **Остановить**
   
   ```bash
   docker compose down
   ```

**Для разработки** в `docker-compose.yml` активирован volume (код монтируется), а в `opcache.ini` включена проверка временных меток, поэтому изменения в `core.php` и `index.php` подхватываются мгновенно без пересборки.

## 🔮 План развития: поддержка текстовых документов (DOCX)

На следующем этапе планируется расширение системы для работы с текстовыми документами формата DOCX. Ключевые задачи:

1. **Расширение модели данных** – добавить поле `doc_type` (image/docx) и адаптировать существующие индексы.
2. **Обработка DOCX** – использовать библиотеку (например, `PhpOffice/PhpWord` или `ZipArchive`) для извлечения текста, метаданных (автор, дата создания, количество страниц).
3. **Хранение контента** – сохранять извлечённый текст в отдельном поле (или в JSONB) для полнотекстового поиска.
4. **Генерация превью** – для DOCX можно создавать миниатюру первой страницы (через конвертацию в изображение) или извлекать обложку.
5. **API-эндпоинты** – добавить новые форматы выдачи (например, `/documents/{id}/text` для получения текста).
6. **Очередь задач** – вынести извлечение текста и генерацию превью в асинхронную очередь на основе PostgreSQL (таблица `jobs`).
7. **Поддержка больших файлов** – для DOCX размером >5 МБ перейти на Large Objects или потоковое чтение.

**Ориентировочный срок:** август-сентябрь 2026.
